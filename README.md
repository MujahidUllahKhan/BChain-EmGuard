# BChain-EmGuard

**Blockchain-Anchored Multi-Sensor Fusion Framework with LLM-Driven Automated Regulatory Notification for Industrial Greenhouse Gas Emission Monitoring**

> Mujahid Ullah Khan Afridi and Hansuk Sohn  
> Department of Industrial Engineering, New Mexico State University  
> Las Cruces, NM 88003, USA  
> mujahida@nmsu.edu

## Overview

BChain-EmGuard is a unified framework that integrates:

1. **Multi-Sensor Fusion** — Redundant heterogeneous sensor arrays (≥3 per gas) with Median Absolute Deviation (MAD) consensus to detect drift, failure, and deliberate tampering
2. **CCTV-Blockchain Anchoring** — Perceptual hash (pHash) fingerprints of camera frames recorded on-chain as tamper-proof visual evidence
3. **Smart Contracts** — Four Solidity 0.8.20 contracts on Hyperledger Besu (IBFT 2.0) recording every reading, anomaly, and alert immutably
4. **LLM Notification Engine** — Natural-language regulatory alerts generated by an LLM and delivered to assigned officers within 45 seconds of anomaly detection

Target gases: CO₂, CH₄, NOₓ, PM₂.₅ (extensible)

## Quick Start

### Smart Contracts

```bash
# Install Node.js dependencies
npm install

# Compile contracts
npx hardhat compile

# Run all tests (26 passing)
npx hardhat test

# Run gas measurement tests
npx hardhat test tests/gas_measurement.test.js

# Deploy to local Hardhat network
npx hardhat run scripts/deploy.js --network hardhat

# Deploy to local Hyperledger Besu node
npx hardhat run scripts/deploy.js --network besu_local
```

### Python Simulation

```bash
# Install Python dependencies
pip install -r requirements.txt

# Step 1: Generate datasets (10 facilities × 4 gases × 30 days)
python3 datasets/generate_datasets.py

# Step 2: Run main anomaly detection simulation
python3 ml/emission_sim.py

# Step 3: Run LLM notification simulation (no API key needed)
python3 ml/llm_notification_sim.py

# Step 4: Gas cost analysis
python3 ml/analyze_gas.py
```

## Dataset Description

Generated by `datasets/generate_datasets.py` using EPA CEMS-grounded emission profiles.

| Parameter | Value |
|---|---|
| Facilities | 10 |
| Gases | CO₂, CH₄, NOₓ, PM₂.₅ |
| Sensors per gas | 3 (heterogeneous types) |
| Simulation period | 30 days |
| Sample interval | 60 seconds |
| Total readings | 1,728,000 |
| Injected anomaly events | ~4 DRIFT + 3 SPIKE + 2 TAMPER + 2 FAILURE + 5 EXCEEDANCE per facility/gas |

Locations cover industrial zones in Pakistan, Bangladesh, UAE, India, Egypt, and Nigeria.

## Attack Vectors Mitigated

| AV | Attack | Mitigation |
|---|---|---|
| AV-1 | Physical sensor obstruction | Multi-sensor consensus + CCTV |
| AV-2 | Clean-air injection | Cross-gas correlation + ML |
| AV-3 | Firmware manipulation | Calibration records + consensus |
| AV-4 | Data interception | HSM signing + signature verification |
| AV-5 | Record alteration | Blockchain immutability |
| AV-6 | Camera destruction | pHash baseline + tamper enclosure |
| AV-7 | Alert flooding | Rate limiting + MAX_ACTIVE_DISPUTES=3 |
